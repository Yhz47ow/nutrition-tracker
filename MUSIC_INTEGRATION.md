# 训练音乐集成可行性报告

调研日期：2026-07-13

## 结论

本项目是纯前端 PWA。浏览器同源策略禁止父页面读取或调用跨域 iframe 内部播放器；Media Session API 也只能声明和控制本页面创建的媒体会话，不能读取其他标签页或手机原生音乐 App 的播放状态。因此不存在一个纯网页方案，可以用自定义按钮同时控制 Apple Music、网易云音乐、QQ音乐和小宇宙原生 App。

最终采用三层方案：

1. Apple Music、网易云音乐使用官方 HTTPS 嵌入播放器，在训练页内完成平台自身支持的播放操作。
2. QQ音乐、小宇宙尝试嵌入官方 Web 页面，并始终提供“在平台打开”降级入口。
3. 本地音频播放列表和离线白噪音由当前页面播放，统一控制条可完整控制上一首、播放/暂停、下一首和独立音量，同时接入 Media Session。

## 平台能力

| 平台 | 可嵌入方式 | 当前结果 | 自定义控制条限制 |
|---|---|---|---|
| Apple Music | `https://embed.music.apple.com/...` | 官方嵌入可加载专辑/歌单；完整播放可能要求登录 | 无公开 iframe 控制协议；MusicKit JS 需要 Apple Developer Token，不适合无后端 PWA |
| 网易云音乐 | `https://music.163.com/outchain/player` | 官方单曲和歌单外链播放器可用 | 无公开 `postMessage` 控制协议，父页面无法读取曲目和音量 |
| QQ音乐 | `https://y.qq.com/...` Web 页面 | 响应头未明确禁止 iframe，但没有稳定的官方外链控制 API | 登录、地区策略或页面改版可能导致嵌入失败，降级为外部打开 |
| 小宇宙 | `https://www.xiaoyuzhoufm.com/...` 单集页 | 响应头未明确禁止 iframe，可尝试加载单集页 | 无跨域播放状态和控制接口，失败时外部打开 |
| 本地音频 | 当前页面 `<audio>` | 完整控制，可加载 URL 列表或本机临时文件 | 远程 URL 是否可播放取决于来源；本机文件刷新后需重新选择 |
| 白噪音 | Web Audio API | 完整控制，离线可用 | 系统省电或浏览器终止页面时会停止 |

2026-07-13 实测响应头：Apple Embed、网易云外链播放器、QQ Web 播放页和小宇宙首页均返回 HTTP 200；Apple 和网易云未设置阻止当前嵌入方式的 `frame-ancestors`/`X-Frame-Options`。平台可随时调整策略，因此 UI 保留外部打开按钮。

## Media Session 与系统 App

`navigator.mediaSession` 的用途是让本页面把曲名、封面和控制处理器暴露给锁屏界面、耳机按键和系统媒体面板。它没有读取其他 App 当前曲目的 API，也不能向另一个标签页发送上一首/下一首命令。本实现仅在本地音频或白噪音播放时注册 Media Session，避免错误接管外部音乐 App。

Web Bluetooth 也不能模拟标准蓝牙耳机的 AVRCP 媒体按键。浏览器没有成为手机蓝牙媒体遥控器所需的系统权限，因此方案 C 不实现；Android 原生容器可以通过系统 MediaSession/通知控制器另行开发。

## 离线行为

音乐控制条、配置界面、白噪音和本地逻辑由 Service Worker 缓存。第三方 iframe 和远程音频不属于本应用资源，离线时不能保证加载；本机已选择的音频文件只在当前页面会话有效。白噪音是唯一完全离线的音源。

## 验证范围

- 自动测试覆盖平台白名单、恶意 URL 拒绝、网易云单曲/歌单参数、Apple Embed URL 转换、本地播放列表解析和配置归一化。
- HTTP 验证覆盖四个平台入口和全部 PWA 本地资源。
- Apple Music 登录播放、QQ音乐登录、网易云版权歌曲和小宇宙 App 跳转受账号、地区、版权和第三方 Cookie 策略影响，必须在用户自己的手机环境验收。
- 统一上一首/播放/下一首/音量控制仅对本地音频和白噪音承诺完整可用；外部平台的控制由嵌入页面自身承担。
